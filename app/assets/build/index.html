What is CoralUI?

CoralUI or CUI (pronounced "cooey") is an implementation of the CloudUI visual style from Adobe XD.

CoralUI's aim is to provide everything your product needs to adopt the CloudUI visual style.

Resolution Independence

CoralUI is resolution independent. By using elastic measurement units to define sizes (em and rem), CoralUI can be scaled to the perfect size for any display, regardless of size or pixel density.

Browser Support

CoralUI supports all modern browsers, including IE 9+, Firefox (latest), Chrome (latest), Safari (latest).

Prevalent mobile devices are also be supported, including Android 3+ and iOS 5+.

Widget Philosophy

All CoralUI widgets are guaranteed to fulfill 8 promises:

  1. Widgets will be available as a jQuery plugin: $('#myModal').modal()
  2. Widgets will be available as a class: myModal = new CUI.Modal({ element: '#myModal' }).
  3. Widget APIs will always be fully documented here.
  4. Widgets will always be easily extendable with CUI.Modal.extend().
  5. Widgets will progressively enhance existing markup, or render a template using passed options if no markup is present.
  6. Widgets will take advantage of declarative markup using data-* attributes.
  7. Widgets will work on both desktop and mobile.
  8. Widgets will be fully tested with automated unit tests.

Icons

Icons can be added to a number of different elements by using the icon-* classes that correspond to the icon's filename.

Usage Guidelines

Icons should be applied to three different tags:

  • <i>: Illustrative
  • <a>: Navigational
  • <button>: Actionable

Accessibility

Always include label inside of the tag to indicate the icon's function. This text isn't visible by default, but is always readable by screen readers.

If you want to show the text, add the withLabel CSS class.

Sizing

When used without context, icons should be given a size:

  • xsmall 12x12
  • small 18x18
  • medium 24x24
  • large 32x32

Variations

Icons will change color when hovered or clicked appropriate based on context. However, you may force a particular color by applying one of the following CSS classes:

  • white White icon at 100% opacity
  • blue Blue icon at 100% opacity

Rail

The side panel used for metadata and feeds.

Positioning

The rail can be placed on the left or the right by applying the left or right class.

Sections

By using the <section> tag, the rail can contain multiple sections separated by a border.

The bare and thin CSS classes can be added to remove the border or padding on each section respectively.

Section headings should be contained within the <h4> tag.

Tabs

A simple markup-driven tab bar used to separate sections of content.

Variations

In addition to the default style, there are 3 variations that can be applied using the following CSS classes:

  • Default tabs: Used in a modal dialog or as sub-tabs
  • nav Page tabs: For when tabs are used to divide content on a page level
  • white White tabs: Used in a modal dialog or as sub-tabs
  • finger Finger tabs: Useful for tabs that have long titles

Additionally you can apply the class full-page in cases where you use the tabs in full-screen situations. This will give them some more margin to not stick to the borders of your screen.

Buttons & Controls

CUI provides many useful styles and widgets for buttons & controls.

Buttons

The <button> tag is styled automatically. You can apply the button CSS class to any element to style it as a button, but be sure to add role="button" for accessibility.

Variations

By default, buttons are gray. There are two CSS classes that can be added to change the look of the button:

  • primary A blue button, used to indicate primary actions such as OK or save
  • warning A red button, used to indicate actions that are potentially destructive, such as delete

Button groups

You can group buttons into horizontal lines by adding a <div class="buttong-group"> around them.

Action

Button groups:

Cards

                
There are many cards for displaying data. They are not always card shaped, but we still call them that.

                
Variations

                
    
                    
  • Project Card: for a project (that contains an asset folder, a website and some collections).
                    
  • Collection Card: for a collection of items.
                    
  • Asset Card: for a single item.
                    
  • Directory Card: for a directory of assets.
                    
  • Page Card: for a (web) page.
                    
  • Board Card: for a Tartan board.
                

                
Usage

                
The markup for a card can be displayed in grid view or in list view, depending if it is wrapped in an element with a grid or with a list class.

                
Some cards can be displayed as a little stack (e.g. pages that contain sub-pages, non-empty collections and non-empty projects). To display these cards as a stack, simply add the stack class to the card.

                
If a card has to be displayed in a list view, it should contain the following elements: <i class="select"></i><i class="move"></i>. If the items cannot be viewed as a list, these elements can be ommitted.

                
To hide some elements in grid or in list view, the show-grid, respectivly the show-list classes can be used.

                
In case an optimized or different image should be displayed in list/grid view, simply place both images into the <span class="image"> with one of the show-grid and show-list class on each of them. There is a  script to avoids that both images to be loaded until the user switches view.

Status Indicators

                
Status indicators provide a way of conveying the progress of an ongoing process or the result of a process that has finished.


                

                    
Alert

                    
Alerts indicate to the result of an operation or that a new information is present.

                    
                    
Variations

                    
There are 5 different styles of alerts:

                    
    
                        
  • error Indicate that an operation has failed or that a critical problem is present
                        
  • notice Indicate that an operation completed with warnings, or that a potential problem is present
                        
  • success Indicate that an operation completed without errors or warnings
                        
  • info Provide information to the user, such as indicating that new reports are available
                        
  • help Provide guidance to the user, such as links to help documentation
                    

                    
                    
Sizing

                    
Alerts usually contain concise messages, but can accommodate larger messages when the large CSS class is added.

                    
                    
The <button class="close"> tag can be omitted if an alert should not be dismissible.

  • success Draw attention to a success state

  • arrow-bottom Point downward
