CUI.CardView

A display of cards that can either be viewed as a grid or a list.

The display mode - grid or list view - can be changed programmatically whenever required.

Grid view has two modes: navigation and selection, which can also be switched programmatically. In navigation mode, the user can use cards to navigate hierarchical structures ("to another stack of cards"). In selection mode, the cards get selected on user interaction instead. List view combines both selection and navigation modes.

The card view uses a data model internally that abstracts the cards. This data model is currently not opened as API. Therefore you will often encounter unspecified objects that represent cards in the data model. You can use them interchangibly (for example, if one method returns a card data object, you can pass it to another method that takes a card data object as a parameter), but you shouldn't assume anything about their internals. You may use CUI.CardView#prepend, CUI.CardView#append and CUI.CardView#removeAllItems to manipulate the data model.

Please note that the current implementation has some limitiations which are documented if known. Subsequent releases of CoralUI will remove those limitations bit by bit.

Defining comparators for column sorting

//  Define a selector for the column and then a comparator to be used for sorting
// The comparator
var comparatorConfig = {".label .main": new CUI.CardView.DefaultComparator(".label h4", null, false),
                   ".label .published": new CUI.CardView.DefaultComparator(".label .published", "data-timestamp", true)};
new CUI.CardView({comparators: comparatorConfig})

Defining comparators via data API

<!-- Page header for list view -->
<header class="card-page selectable movable">
    <i class="select"></i>
    <i class="sort"></i>
    <div class="label">
        <div class="main" data-sort-selector=".label h4">Title</div>
        <div class="published" data-sort-selector=".label .published .date" data-sort-attribute="data-timestamp" data-sort-type="numeric">Published</div>
        <div class="modified" data-sort-selector=".label .modified .date" data-sort-attribute="data-timestamp" data-sort-type="numeric">Modified</div>
        <div class="links" data-sort-selector=".label .links-number" data-sort-type="numeric"><i class="icon-arrowright">Links</i></div>
    </div>
</header>
<!--
    Sorting is started when the user clicks on the corresponding column header.

    data-sort-selector   defines which part of the item to select for sorting
    data-sort-attribute  defines which attribute of the selected item element should be user for sorting. If not given, the inner text is used.
    data-sort-type       if set to "numeric", a numerical comparison is used for sorting, an alphabetical otherwise
-->

CUI.CardView

A display of cards that can either be viewed as a grid or a list.

The display mode - grid or list view - can be changed programmatically whenever required.

Grid view has two modes: navigation and selection, which can also be switched programmatically. In navigation mode, the user can use cards to navigate hierarchical structures ("to another stack of cards"). In selection mode, the cards get selected on user interaction instead. List view combines both selection and navigation modes.

The card view uses a data model internally that abstracts the cards. This data model is currently not opened as API. Therefore you will often encounter unspecified objects that represent cards in the data model. You can use them interchangibly (for example, if one method returns a card data object, you can pass it to another method that takes a card data object as a parameter), but you shouldn't assume anything about their internals. You may use CUI.CardView#prepend, CUI.CardView#append and CUI.CardView#removeAllItems to manipulate the data model.

Please note that the current implementation has some limitiations which are documented if known. Subsequent releases of CoralUI will remove those limitations bit by bit.
